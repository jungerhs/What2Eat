# 类别标签规范 (Label Guidelines)

本文件是 query 分类任务的**唯一权威契约**，所有数据合成、人工校验、二次打标均以此为准。

任务：将用户的烹饪类问句分为 4 类：`general` / `detail` / `multi-hop` / `recommend`。

---

## 类别定义

### 1. `general` — 概念性/整菜概览

**特征**：单实体 + 宽泛动词，期望拿到**整道菜/整类菜**的概览、做法、由来、文化背景。

**强信号**：
- 动词：「怎么做 / 是什么 / 介绍一下 / 讲讲 / 说说 / 教我做」
- 单实体（菜名 / 菜系名 / 食材名）
- 概念性疑问（"什么是 X"、"X 怎么做"、"X 有哪些"）

**典型正例**：
- 红烧肉怎么做
- 介绍一下糖醋口味
- 川菜是什么
- 麻婆豆腐是哪里的菜
- 凉拌菜一般怎么做

**反例（不要标 general）**：
- 红烧肉要炖多久 → **detail**（时间疑问）
- 用鸡蛋和番茄做什么菜 → **multi-hop**（多实体）
- 推荐一道川菜 → **recommend**（推荐意图）

---

### 2. `detail` — 单点细节

**特征**：期望**精确数值/具体步骤/比例/火候**等单点信息。

**强信号**：
- 数字疑问：多少 / 多久 / 几勺 / 几片 / 几分钟 / 几度
- 比例 / 火候 / 温度 / 时间
- 「第几步 / 什么时候放 / 先放什么后放什么 / 为什么这一步」
- 「用什么替代 / 能不能不放」

**典型正例**：
- 红烧肉要炖多久
- 冰糖和酱油的比例是多少
- 第 3 步是什么意思
- 糖醋汁什么时候放
- 没有料酒可以替代吗
- 麻婆豆腐要小火还是大火

**反例**：
- 红烧肉怎么做 → **general**
- 红烧肉和糖醋排骨哪个好吃 → **multi-hop**（多实体比较）

---

### 3. `multi-hop` — 多跳推理 / 多实体组合

**特征**：含 **≥2 个实体** 或 **显式过滤条件**，需要组合信息才能回答。

**强信号**：
- 多个实体并列：「X 和 Y 一起 / 搭配 / 组合 / 配什么 / 还有哪些」
- 过滤条件：「适合 / 不含 / 只要 / 只要 / 排除 / 限定（早餐/低脂/夏/冬/新手/快手）」
- 跨类别查询：「X 菜系里用 Y 食材的菜」
- 比较 / 关系：「X 和 Y 的区别 / 哪个 / 关系」

**典型正例**：
- 用鸡蛋和番茄可以做什么菜
- 川菜中适合早餐的素菜
- 红烧肉和糖醋排骨哪个更下饭
- 减脂期吃什么水产
- 不含花生的川菜有哪些
- 早餐适合的汤类有哪些

**反例**：
- 推荐几道川菜 → **recommend**（单类 + 场景，仍是推荐意图）
- 红烧肉要炖多久 → **detail**（单实体单点）

---

### 4. `recommend` — 显式推荐

**特征**：用户**主动请求推荐**，期望得到**多道菜的列表**。

**强信号**：
- 「推荐 / 介绍几道 / 有什么 / 来点 / 来一份 + 菜 / 来个 + 菜」
- 「适合 X 的 + 名词」
- 求清单、求点单

**典型正例**：
- 推荐一道川菜
- 有什么简单易做的早餐
- 介绍几道下饭菜
- 夏天适合吃什么凉菜
- 来个简单的家常菜
- 有哪些适合新手的菜

**反例**：
- 川菜是什么 → **general**（无推荐动词）
- 用鸡蛋和番茄可以做什么菜 → **multi-hop**（多实体组合，不是"求清单"）

---

## 边界对照表（最容易判错）

| query | 类别 | 关键判别点 |
|---|---|---|
| 红烧肉怎么做 | general | 单实体 + 宽泛动词 |
| 红烧肉要炖多久 | detail | 数字疑问 |
| 红烧肉和糖醋排骨的区别 | multi-hop | 多实体 + 比较 |
| 推荐红烧肉和糖醋排骨 | recommend | 推荐 + 多菜 |
| 鸡蛋和番茄能做什么 | multi-hop | 多实体 + 组合 |
| 鸡蛋和番茄推荐做法 | recommend | "推荐" 词覆盖 |
| 适合早餐的菜 | recommend | "适合 + 场景" 模板 |
| 适合早餐的汤类有哪些 | multi-hop | "有哪些" + 过滤 |
| 川菜是什么 | general | 概念性 |
| 介绍几道川菜 | recommend | "介绍几道" 推荐动词 |
| 红烧肉放多少盐 | detail | 数字疑问 |
| 红烧肉可以用什么代替冰糖 | detail | 替代疑问（单点） |
| 红烧肉和什么搭配好吃 | multi-hop | 搭配 + 单实体（推荐搭配也算多跳） |

---

## 判别流程（给 LLM 二次打标用）

按以下顺序判断，命中即归类：

1. 含「推荐 / 介绍几道 / 有什么 / 来点 + 名词」 → **recommend**
2. 含 **≥2 实体** 或 **过滤词（适合/不含/只要/低脂/早餐/快手/新手/夏/冬 + 类别词）** 或 **比较（哪个/区别）** → **multi-hop**
3. 含 **数字疑问**（多少/多久/几/比例/火候/温度/第几步/什么时候） → **detail**
4. 其它 → **general**

注意：第 1、2 步的优先级最高；第 3 步在单实体情况下才稳；第 4 步兜底。
